Safe Traversal Operator '?'

aka Safe Navigation

When rendering complex objects some of the attributes/properties of the object may not exist at the moment the DOM is initialized
Adding the '?' before the dot allows the value to be null without adversely impacting the rendering.

{ { task.assignee?.name } } = {{ task.assignee?.name }}
{ { task.assignee?.ID } } = {{ task.assignee?.ID }}

When the assignee is null, the 'Cannot read property 'name' of null' error will occur and the DOM will not be updated past this point.
{ { task.assignee.name } } = {{ task.assignee.name }}
{ { task.assignee.ID } } = {{ task.assignee.ID }}
